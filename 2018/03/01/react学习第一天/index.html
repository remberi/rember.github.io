react学习第一天

作者:巅峰の大香蕉    发布于: 2018-03-01

之前关注react很久了，一直想要去实践尝试，但苦于没有充足的时间去研究，都是一天看一点点，也是一只半解，只知道react是一个视图框架，facebook提出，前端模块化先驱，单项数据流提倡者…而且学习成本比vue高，比较适合大型前端项目。。。。。。。。。

在SegmentFault找到一篇比较适合入门级的教程，因为之前学过vue，所以一直很好奇react到底和vue有什么区别，开头介绍了react的发展情况，主要是为了解决前端繁琐重复的造轮子，解放前端视图更新，将数据和视图关联起来，这么一听和vue好像差不多，vue也是数据驱动模型，不同的是react仅负责视图，vue是mvvm(模型+视图)

react 定义组件的写法有两种，一个是类继承，一种是函数式；（vue是函数式，分7种沃德天怎么辣么多【字符串、模板字面量、x-template、内联模板、render 函数、JSF、单文件组件）目前单文件组件用的比较多，可能和它能把资源和模版独立开来有关吧

react类继承写法

import React, {Component} from 'react';
class HelloMessage extends Component {
    render() {
        return <div>Hello world.</div>;
    }
}

学过java的同学是不是感觉很熟悉，关键字class，extend玛德这个不就是java里面的继承吗!!!(ES6 为 JavaScript 增加了类和类继承的特性。子类会继承父类的“基因”（成员方法、属性），如果父类是一个组件，那子类自然而然也是一个组件)。如果有后端基础的同学看到react的写法一定会感的很亲切=。=[还好之前在培训班耍过几天java]

react 函数式写法
import React from ‘react’;

function HelloMessage() {
    return <div>Hello world.</div>;
}

// 或者：

const HelloMessage = () => <div>Hello world.</div>;

注意一下啊！组件的第一个字母必须大写，不然react会因为小写看错成内部标签鸡儿不渲染组件的。

目前继承式写法比较多。。。。。因为

类继承的方式，相较于函数式组件，虽然写起来略繁琐，但是它拥有更多的特性：内部状态：state，生命周期函数

函数式组件虽然没有 state 和生命周期函数等特性，但是它有更简洁的书写方式，另外还有更好的性能，不用处理一些复杂的特性，执行效率当然高了。

react 生命周期

componentWillMount：在组件被渲染到页面上之前执行，在组件的整个生命周期内只执行一次。在这里可以调用 setState 更新内部状态，但是更推荐将这里的状态更新操作放到 constructor 中。该函数执行完后会立马执行 render 方法并将组件渲染到页面上。所以，在这里执行 setState 不会触发额外的渲染过程，因为这是没有必要的。

componentDidMount：组件被渲染到页面上后立马执行，在组件的整个生命周期内只执行一次。这个时候是做如下操作的好时机：
        某些依赖组件 DOM 节点的操作。
        发起网络请求。
        设置 setInterval、setTimeout 等计时器操作。
        在这里可以调用 setState 更新组件内部状态，且会触发一个重新渲染的过程，即会重新执行 render 方法并更新视图。

componentWillReceiveProps：componentWillReceiveProps(nextProps)
                        该声明周期函数可能在两种情况下被调用：

                        组件接收到了新的属性。新的属性会通过 nextProps 获取到。

                        组件没有收到新的属性，但是由于父组件重新渲染导致当前组件也被重新渲染。

                        你只要知道，当该函数被调用时，并不一定是因为属性发生了变化。

                        在这里也可以调用 setState 更新组件的内部状态，同样也不会触发额外的重新渲染操作，React 会聪明地用更新后的属性和内部状态进行一次重新渲染。

react：状态提升

状态提升的意思是，当组件 A 需要依赖另外一个组件 B 的内部状态，而他们又不是父子关系时，需要将组件 B 的内部状态提升到他们公共的祖先组件中管理。这样他们就都可以通过属性接收到这份数据了。
当组件 B 需要对数据进行变更时，可以通过函数属性来通知祖先组件对数据更新，然后重新传递给子组件。


react中遵从单项数据流的时候存在的问题，当页面逻辑层级非常深的时候，组件之间通信就变的非常的繁琐，虽然有替代方案及ref实现但是这个不鼓励而且以后的版本更新可能需去除。
